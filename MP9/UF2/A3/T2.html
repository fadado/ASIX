Publicació a Internet dels documents generats

GitHub Pages

GitHub ofereix un mitjà fantàstic per publicar informació estàtica a Internet: GitHub Pages. Cada usuari pot crear un repositori, amb un nom predeterminat, que serà publicat automàticament a Internet en el domini github.io. Podem usar HTML, CSS, JavaScript, imatges… qualsevol tipus de fitxer estàtic!

Enllaços recomanats

  • GitHub Pages
  • Ajuda sobre GitHub Pages

Pràctiques I

El tutorial de GitHub pàges és simple i breu. Ara el practicarem, però amb la idea d’esborrar-lo per començar després de nou.

  • Seguint les explicacions de GitHub Pages crea un lloc web estàtic.
  • Experimenta afegint continguts de prova al teu repositori i publica’ls simplement fent git push.

Ara has de gestionar dos repositoris: el que té els documents font i el de publicació. Per començar còpia manualment el fitxers XHTML generats en el primer repositori dins del segon.

  • Elimina i crea de nou el teu repositori per publicar a GitHub Pages .
  • Publica en el nou repositori els documents generats en activitats anteriors a partir de la documentació de MarkDown.
  • Afegeix al teu lloc una pàgina principal (index.html), més pàgines, CSS, imatges…

Pràctiques II

Una vegada implementat el sistema de publicació podem millorar el repositori on tenim ls documents font. Aquesta activitat pot no tenir fi, i aquesta pràctica és tan sols un exemple de per on començar… En la propera activitat usarem Jekyll en lloc reinventar la roda, pel que aquesta pràctica tan sols vol fer-te reflexionar sobre les dimensions del problema.

  • Automatitza amb un script la còpia dels fitxers generats cap el repositori de publicació: cp -u ..., rsync -a ..., etc.
  • Millora la gestió dels documents situant la seva metainformació en fitxers externs (esborra l’antiga metainformació incrustada dins del document). Per cada fitxer amb extensió .md crea un amb el mateix nom i extensió .yaml, i modifica el script que genera la documentació per suportar aquest disseny.
  • Prepara un fitxer amb metainformació comuna a tots els documents, i adapta la plantilla per que en faci us.
  • Com faries per gestionar l’estructura general del lloc web amb Genshi (en lloc d’usar les plantilles de Pandoc) integrant els continguts MarkDown transformats per Pandoc? On posaries la metainformació?
