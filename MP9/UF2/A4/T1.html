Instal·lació i prova de Jekyll

Generadors estàtics de llocs web

Actualment estem vivint un renéixer molt fort dels generadors estàtics de llocs web, i de proveïdors de serveis per allotjar-los. En coherència amb tot els vist al llarg del curs, dedicarem aquest temps a estudiar gestors de continguts que treballen amb aquesta filosofia.

Jekyll

El gestor de continguts que estudiarem és Jekyll, que començarem a usar de forma independent, però que més endavant veurem que està integrat perfectament amb GitHub Pages.

Important: Aquesta activitat t’exigirà un alt grau d’autonomia. Pots imaginar que es tracta d’una mena de mini-projecte, similar en la forma de treballar al projecte final del cicle.

El sistema recomanat per GitHub per preparar els llocs web de GitHub Pages és Jekyll, i si ells ho diuen per alguna cosa serà… Anem a explorar! Per estalviar-te feina inicial, configura el teu sistema amb aquests paquets:

	sudo yum install rubygems rubygems-devel ruby-devel
	# si uses Fedora 23 també cal:
	sudo dnf install redhat-rpm-config
	# finalment
	sudo gem install jekyll

Poden fer falta altres paquets, com el compilador de C (sudo yum install gcc), etc., però el més probable és que els tinguis instal·lats.

Pràctiques

  • Segueix detalladament el tutorial Jekyll. El resultat ha de ser un lloc web local, que pots personalitzar al teu gust.
  • Crea i explora un nou lloc executant aquestes ordres (per aturar el servidor et caldrà usar ps i kill): 
    	jekyll new LLOC
	cd LLOC
	jekyll serve --detach
	firefox http://localhost:4000 & disown
    Cal que puguis explicar la funció de qualsevol dels fitxers font del lloc!
  • Modifica els continguts del nou lloc per que estiguin adaptats i personalitzats a tu mateix. Executa jekyll d’aquesta forma per poder observar els canvis a mesura que els facis: 
    	jekyll serve --watch
    Pensa que si fas canvis a _config.yml has de reiniciar el servidor.
